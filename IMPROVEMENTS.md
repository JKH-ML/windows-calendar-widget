## [2차] 프로덕션 배포 대비 개선 사항

### 9. [x] 자동 업데이트 (Auto-update)
**문제:** 배포 후 버그 수정/기능 추가 시 사용자가 수동으로 exe를 교체해야 함.  
**해결:** GitHub Releases를 활용한 자동 업데이트 구현.  
- 앱 시작 시 GitHub API로 최신 릴리즈 태그 조회
- 현재 버전보다 높으면 UI에 배너/알림 표시
- 다운로드 링크를 브라우저로 열어줌 (직접 교체 유도)

---

### 10. [x] 오류가 사용자에게 안 보임 (Silent failure)
**파일:** `frontend/src/components/calendar-demo.tsx`  
**문제:** `syncAndLoad()`가 실패해도 `console.warn`만 출력. 사용자는 동기화가 됐는지 실패했는지 알 수 없음.  
**해결:** 동기화 실패 시 UI 상단에 에러 배너(토스트) 표시.

---

### 11. [x] `wails.json` 작성자 정보 수정
**파일:** `wails.json`  
**문제:** `"author": { "name": "mahcks", "email": "max@mahcks.com" }` — 원본 템플릿의 타인 정보가 그대로.  
**해결:** 실제 작성자 정보로 교체.

---

### 12. [x] 뷰 전환 UI 버튼 확인 및 추가
**문제:** `setMode` 연결은 됐지만 헤더에 월/주/일 전환 버튼이 실제로 보이는지 확인 필요. 없으면 추가.  
**해결:** 헤더에 뷰 전환 버튼(월/주/일) 추가.

---

### 13. [ ] 반복 일정(RRULE) 프론트엔드 전개
**문제:** 반복 일정이 DB에 저장되지만 프론트에서 날짜별로 펼쳐 보여주지 않음. 월간 뷰에서 반복 일정이 한 번만 표시됨.  
**해결:** `rrule` 라이브러리로 이벤트 목록 전처리 → 뷰 범위 내 반복 인스턴스 생성.

---

### 14. [ ] 알림(Alert) 기능 실제 구현
**문제:** `alert` 필드가 UI/DB에 있지만 실제 알림이 발생하지 않음.  
**해결:** Go 백엔드에서 알림 시간 계산 후 Windows 알림(`wailsruntime` 이벤트 또는 tray notification) 발송.
